1.3. sqlserver数据库创建说明

使用以下代码创建数据库以及表格。

CREATE DATABASE WebTable;
GO
use Webtable;
CREATE TABLE pendingUrl(
[ID] BIGINT PRIMARY KEY IDENTITY(1,1),
[url] NTEXT NOT NULL
)

CREATE TABLE personUrlAndHtml(
[ID] BIGINT PRIMARY KEY IDENTITY(1,1),
[url] TEXT NOT NULL,
[html] NTEXT NOT NULL
)

CREATE TABLE entityAndRelationship(
[ID] BIGINT PRIMARY KEY IDENTITY(1,1),
[entity] NTEXT,
[relationship] NTEXT
)

CREATE TABLE uselessUrl(
[ID] BIGINT PRIMARY KEY IDENTITY(1,1),
[url] NTEXT NOT NULL
)

GO

以上语句会创建4个数据表。

其中pendingUrl是指待处理的URL队列，该表中的URL链接必定是人物相关的链接。

而uselessUrl是指无用的URL链接，该表中的链接是与人物无关的链接。

personUrlAndHtml中记录的是url和当前url对应的html串，接下来表格信息抽取系统，会对这个html串抽取其中的表格。

而entityAndRelationship则用于存储实体和关系，其中entity是实体，relationship是关系。这其中，ID是BIGINT类型，其余的全部都是NTEXT类型。

4. 原理简介

4.1. 总述

该系统需要完成从网页中抽取表格，提取有效数据并将数据存储到图数据库的全过程。

系统的整体结构图如图6所示。本文从整体上分为四部分：（1）爬虫系统。（2）表格信息抽取系统。（3）知识存储系统。（4）可视化系统。

图 6： 系统整体结构图

4.2. 爬虫系统

爬虫系统需要保证稳定健壮的网页爬取工作，还需要识别出与主题有关的Web页面。为了方便获取高质量数据，这里选用了百度百科作为爬取对象。该爬虫系统的设计目标是可以遍历整个百度百科网页，获取其中的表格，并以此获取之中的人物信息。爬虫系统流程图如图7所示。

爬虫系统运行步骤如下：

（1）获取种子链接。 种子链接是指初始时指定的一批待爬取URL链接。该爬虫系统可以从种子链接出发，逐渐遍历整个网络。因为抽取的主题是“人物”，因此种子链接中最好包含大量的人物索引。例如：https://baike.baidu.com/item/中国名人。 将种子链接读取到“待爬取队列”，等待爬取循环开始。

（2）从“待爬取队列”中取出一条URL，读取该URL对应的Web页面，抽取该页面的html（超文本标记语言），并从中抽取与人物有关的URL链接，将这些链接存入SQLSERVER数据库中的“有效URL数据表”，将与人物无关的URL链接存入到“无效URL数据表”，之后再读取一条URL，进入循环。

（3）读取“有效URL数据表”，将其中的URL链接写入到“待爬取队列”，完成一个循环。以上（2）（3）步骤完成了BFS（广度优先）遍历。

（4）新建立一个线程，用以处理无效的URL。该线程从“无效URL数据表”中读取一条URL链接，爬取URL对应的Web页面，获取html后，从中抽取与人物有关的URL链接，将这些链接存入SQLSERVER数据库中的“有效URL数据表”，将与人物无关的URL链接存入到“无效URL数据表”，之后再读取一条URL，进入循环。

图 7： 爬虫系统流程图

4.4. 知识存储系统

抽取实体与关系信息后，需要进行存储。

本文将获取的实体与关系存储到neo4j数据库中。抽取的表格虽然来自于不同网页，但其中的关系可能会指向相同的人物，因此需要进行实体对齐。实体对齐是指判别两个实体是否相同的过程。本文中的实体对齐方法如下：

1、 若人物包含url链接，且url链接与数据库中某一人物的url链接相同，则实体融合。若无相同链接，但与数据中某一个链接存在相互包含的关系，例如“https://baike.baidu.com/item/严夏”包含于“https://baike.baidu.com/item/夏言/4270379”，则也可进行实体融合。若不包含url链接，则进行下一步。

2、 若人名以“氏”结尾，则必然分开存储。例如：刘氏、李氏。否则进入下一步。

5.1.2. getTable() 函数

    def getTable(self, _html: str) -> list:
        """
        表格定位，从html中抽取表格，并返回一个list，该list中每个元素都是一个Table对象
        :param _html:待抽取的html串
        :return:一个list，该list中每个元素都是一个Table对象
        """
        # 预处理
        _html, _soup = htmlPreTreat(_html)  # html预处理
        _tableList = self.extractNonstandardTable(_soup)  # 抽取非规范表，也就是百度百科中的“基本信息”(basic info)列表
        tagTable = _soup.find_all(name="table")  # 找到所有的Table标签

        for tag in tagTable:
            # 首先判断是否为非规范表，这种非规范表带有<table>标签，内部却是ul标签，需要当初考虑和处理
            tableList = self.extractListTable(tag)
            if len(tableList) > 0:
                _tableList.extend(tableList)
            elif self.throughHeuristicRule(tag):  # 看是否能通过启发式规则
                caption = getCaption(tag)  # 获取表格的标题
                aTable = changeTig2Table(tag, caption)  # 将标签转化为Table对象
                _tableList.append(aTable)
        return _tableList

该函数首先使用htmlPreTreat函数对html进行预处理，并且将其转化为DOM树，其中变量_soup就是这个DOM树。

之后使用extractNonstandardTable函数，用于从Web页面中抽取“非规范表格”， 一般是标签类名中带有basic info的列表，如图所示：

图 9： basic info列表

接下来使用find_all函数寻找到所有标签名带有<table>的标签，得到一个列表，该列表中每个元素都是<table>标签的对象。使用for循环遍历这个列表，首先判断该<table>标签是否为“非规范表”————有一些表格虽然带有<table>标签，但是内部结构却是<ul>或者<li>标签的列表，所以这种情况需要单独考虑转化。这种表如图10所示。使用extractListTable函数来抽取这些非规范表。

图10： 特殊的带有<table>标签的非规范表

在抽取完非规范表后，需要抽取普通表。这时候首先用throughHeuristicRule函数判断是否能通过3条启发式规则，如果全都能通过，则说明该表格为有效表格，进行下一步判断。

接下来使用getCaption函数获取当前表格的标题，并且使用changeTig2Table函数，将当前的标签转化为Table类对象，最终将该对象加入到列表中，并返回列表。

5.1.3. dealWithTableList() 函数

        def dealWithTableList(self, _tableList: list):
        """
        处理Table对象的list，将这些表格中的实体和关系抽取出来，之后写下记录文件，便于之后的动态可视化处理
        将“表格、实体、关系、关系图”全都写入到本地文件中
        :param _tableList: Table的列表，该列表中每个元素都是一个Table对象
        :return:
        """
        entityListAndRelationshipList = []
        # 将当前表格深度复制一份，便于之后的删除行列的操作
        _extendTableList = deepcopy(_tableList)
        for table in _extendTableList:
            table.hrefMap[self.nowName] = self.nowUrl  # 超链接映射
            table = table.extendTable()  # 表格规整
            table.prefix = self.nowName  # 设置表格的前驱
            if table.isNormal() and table.isCorrect():  # 判断表格是否正常且正确
                if table.getUnfoldDirection() == "COL":  # 把表格全部变成横向展开的
                    table = table.flip()
                table.clearTable()  # 清理表格数据
                entity, relationship = table.extractEntityRelationship()  # 抽取实体和关系
                # 若存在实体和关系，则将其写入到数据库之中
                if len(entity) != 0 or len(relationship) != 0:
                    entityJson = json.dumps(entity, ensure_ascii=False)
                    relationshipJson = json.dumps(relationship, ensure_ascii=False)
                    entityListAndRelationshipList.append([entity, relationship])
                    self.sql.writeER2DB('entityAndRelationship', entityJson, relationshipJson)
        # 将数据记录到本地文件，方便可视化处理
        self.writeRecords(self.nowUrl, _tableList, entityListAndRelationshipList)

得到表格后，需要进行后序的处理。

首先使用deepcopy函数深度复制一份当前表格，因为在后续的“关系抽取”中可能会删除行或者删除列，为了不影响原来的表格，这里复制了一份。

接下来使用extendTable函数，将表格规整。所谓表格规整，就是指如图11所示由上表变为下表的过程。

图11： 表格规整

接下来判断表格是否正常：即行列数均大于等于2？

再判断表格是否正确：即行列单元数相同？

如果既正常又正确，那么需要使用getUnfoldDirection函数获取表格的展开方向，如果当前表格是按列展开(纵向展开)的，那么需要使用filp函数将其翻转为按行展开（横向展开）的。表格的展开方向有三种：按行展开、按列展开、按照行列同时展开。可以从图12中看到这三种方向的表格。默认第三种不考虑，因为这种表格太少。

图12： 3种展开方向的表格

接下来使用extractEntityRelationship函数从表格中抽取对应的实体和关系。如果存在实体或者关系，则通过josn.dumps函数将其转化为JSON串，再通过writeER2DB函数将实体和关系串存储到sqlserver数据库，便于后续的处理。

最后使用writeRecords函数记录当前的表格、实体和关系，这个操作是为了把这些数据写入到pkl文件中，便于之后的可视化显示界面。

            relationshipList = [data[1] for data in entityAndRelationshipList]
            # 实体转化为json串
            entityJson = json.dumps(entityList, sort_keys=True, indent=4, separators=(',', ':'), ensure_ascii=False)
            # 噶虚拟转化为json串
            relationshipJson = json.dumps(relationshipList, sort_keys=True, indent=4, separators=(',', ':'),
                                          ensure_ascii=False)
            # 可视化界面中，实体显示区，设置文本
            self.entityEditor.setPlainText(entityJson)
            # 可视化界面中，关系显示区，设置文本
            self.relationshipEditor.setPlainText(relationshipJson)
            # 将表格布局清空，便于放置新的表格
            clearLayOut(self.tableLayout)
            _tableSplitter = QSplitter(Qt.Vertical)
            tabWidget = QTabWidget()
            for _table in tableList:
                addTable(tabWidget, _table)
            _tableSplitter.addWidget(tabWidget)
            self.tableLayout.addWidget(_tableSplitter)
            # 显示关系图
            self.showRelationChart(htmlPath)
        else:
            QMessageBox.warning(self, "标题", f"未找到该人物", QMessageBox.Yes)

该函数较为复杂，但理解起来很简单。该函数用于显示url对应的表格和关系图，其内部有两个子函数，分别是getDataListWithUrl(url: str)和addTable(tabWidget: QTabWidget, table: Table)函数。

getDataListWithUrl(url: str)中，首先根据url，从本地的字典文件中查询到该url所对应的“ID名”，ID名也就是存入到本地的不带后缀的文件名，因为表格list存入到pkl文件，而关系图数据存入到html文件。故两个文件的名称分别是{ID名}.pkl和{ID名}.html。

addTable(tabWidget: QTabWidget, table: Table)是将当前的表格添加到可视化页面的表格控件之中。

5.2. 运行爬虫

点击菜单栏中“运行”，即可看到“运行爬虫”按钮，点击后，爬虫系统自动运行。因为“运行爬虫”绑定了WebSpiderThread对象中的start函数。其代码如下所示：

class WebSpiderThread(QThread):
    SLockSignal = pyqtSignal()  # 锁信号量，点击后直到爬虫停止，无法点击第二次
    SUnLockSignal = pyqtSignal()  # 解锁信号，程序停止后，自动解开锁

    def __init__(self):
        super().__init__()
        self.webSpider = WebSpider()  # 爬虫程序

    def run(self):
        self.SLockSignal.emit()
        self.webSpider.start(2, float('inf'))  # 2个线程的爬虫，爬取次数为无限次
        while self.webSpider.running: # 线程进入循环
            time.sleep(0.5)
        self.SUnLockSignal.emit()

这个函数中start函数为关键。其函数说明如下：

    def start(self, threadsNum: int = 1, maxCount=10000, IsDealWithSeed=True):
        """
        开启爬虫
        :param IsDealWithSeed: 是否处理种子文件
        :param threadsNum:线程数
        :param maxCount:最大爬取次数
        :return:无
        """
        self.running = True
        requests.DEFAULT_RETRIES = 0  # 重试连接次数
        s = requests.session()
        s.keep_alive = False  # 关闭多余连接
        self.maxCount = maxCount
        # 读取种子链接到待爬队列
        if IsDealWithSeed:
            self.dealWithSeed() # 处理种子链接
        for ID in range(1, threadsNum + 1):
            threading.Thread(target=self.startSpider, args=(ID,)).start()
        threading.Thread(target=self.writeQueue2Database).start()
        threading.Thread(target=self.dealWithUselessUrl).start()

在WebSpider.start函数的末尾，单独开启了两个线程。这两个线程内包含两个函数：writeQueue2Database和dealWithUselessUrl函数。

爬虫运行时，需要爬取url，得到相应的html串，之后这些url和html串需要存储起来，便于之后的“表格信息抽取系统”进行关系和实体抽取。

因为有多个线程的爬虫，为了让数据库不发生读写出错，这里将这些待写入的url和html先存入到一个队列中。每隔一段时间读取队列，判断队列中是否含有数据，若有，则存到数据库。若没有，则继续等待。实现这一功能的，正是writeQueue2Database函数。

而dealWithUselessUrl函数用于处理那些被认为与人物无关的url链接，从中抽取出html，再从html中抽取url链接，按照是否与人物有关，存储到有效url数据表和无效url数据表，以此完成循环，可以让爬虫源源不断的抽取。理论上可以遍历整个百度百科。

class KnowGraphThread(QThread):
    """
    构建知识图谱的线程

run函数中调用了PersonGraph对象的start函数。该函数代码如下：

    def start(self, maxWaitTimes=float('inf')):
        """
        开始构建知识图谱
        :param maxWaitTimes:
        :return:
        """
        self.running = True
        print("开始构建知识图谱...")
        pendingQueue = Queue(maxsize=200)  # 数据队列
        waitTimes = maxWaitTimes
        while waitTimes:
            if not self.running:
                return
            self.addQueue(pendingQueue, 'entityAndRelationship')
            while not pendingQueue.empty():
                if not self.running:
                    return
                waitTimes = maxWaitTimes
                ER = pendingQueue.get_nowait()
                entityList = json.loads(ER[0])
                relationshipTriadList = json.loads(ER[1])
                if entityList is not None and len(entityList) != 0:
                    self.__createNodeWithEntityList("person", entityList)  # 创建节点
                if relationshipTriadList is not None and len(relationshipTriadList) != 0:
                    self.__creteRelationshipsWithList(relationshipTriadList)  # 创建关系

                self.addQueue(pendingQueue, 'entityAndRelationship')

            time.sleep(0.2)
            waitTimes -= 1

该函数中最重要的是两个函数createNodeWithEntityList() 和createRelationshipWithList()函数，分别用于创建节点和创建关系。

__createNodeWithEntityList()函数

该方法采用了“无则创建，有则融合”的策略。根据url链接、标签、属性列表，来判断当前数据库中是否存在此节点，如果没有，则创建。如果有，则进行融合。特殊情况，当人名的末尾以“氏”结尾时，默认创建。

__createRelationshipWithList()函数

该方法将传入的三元组列表转化为关系。其代码如下：

    def __creteRelationshipsWithList(self, triadList: list):
        """
        用三元组列表创建多条关系
        :param triadList: 三元组列表，例如[['小王'，'爸爸'，'老王'],['小白','哥哥','大白']]
        :return: 无
        """
        for triad in triadList:
            node1 = self.__getNode("person", triad[0][0], triad[0][1])
            relationship = triad[1]
            node2 = self.__getNode("person", triad[2][0], triad[2][1])
            self.__createRelationship(node1, relationship, node2)

其中__getNode()函数用于从数据库中获取，是否含有指定的节点，如果有，则返回，如果没有，则创建。
